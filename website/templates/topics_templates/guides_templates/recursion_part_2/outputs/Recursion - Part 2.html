In Order Traversal

Problem Statement:
Given the root of a binary tree, return the inorder traversal of its nodes' values.

What do we know:

  • In order traversal starts from the left side, visits the parent and then the right child.
  • In other words,we need to first visit, the most left child, then get back at its parent and then the right child of the parent.
  • Base case: If the root is empty, we need to return.
  • Recursive case: We visit the left child, then we take the parent's value and then we visit the right child.
  • Basically, the stack will increase as long as there are left children in which case, the stack will start popping off frames. The right children will rebuilt the stack, after which the function returns.
  • Visualising the code below it will be something like this: Frame 1 is called with the value 0. Frame 1 creates Frame 2 with the value 5. Frame 2 creates Frame 3 with the value 3. Frame 3 creates Frame 4 which terminates. Frame 3 appends its value and creates Frame 4, which terminates again (because the node does not have any children). Frame 3 terminates, Frame 2 appends its value and creates Frame 3 with the value 4. And so on until all of the tree's nodes are visited.

Subsets Of An Array

Problem Statement:
Given an array, print all possible unique subsets

What do we know:

  • We need a way to not generate duplicate subsets.
  • We need a way to account for all possible combinations in a list.
  • Base case: The list is empty.
  • Recursive case: As long as there are elements in the list, we can keep adding elements to the set.
  • Visualising the code below, we can think of the problem like this: Frame 1 is called with the element 54 at index 0. Frame 2 is called with the element 49 at index 1, and the path is now [54]. Frame 3 is called with the element 2 and the path is [54,49]. Frame 4 is created with [54,49,2] and cannot create another frame because of the for loop. Frame 3 does not have any more elements to traverse, hence it terminates. In Frame 2, however, the for loop is now at index = 2 (our i variable), hence it creates again the Frame 3 with the path set at [54,2]. And so on.
  • The if condition prohibits us from creating duplicate sets, as if an element is equal to its previous one (in a sorted array), we will have already created all of the possible sets.

Combination Sum

Problem Statement:
Given an array of positive integers arr[] and an integer x, The task is to find all unique combinations in arr[] where the sum is equal to x. The same repeated number may be chosen from arr[] an unlimited number of times. Elements in a combination (a1, a2, …, ak) must be printed in non-descending order. (ie, a1 <= a2 <= … <= ak).

What do we know:

  • A number can be used unlimited number of times.
  • We need to build a sum with the given numbers that will amount to our target.
  • Base case: We pass the target, so we terminate.
  • Recursive case: We are below our target, so we keep adding numbers, either the same one or the next one in the list.
  • Visualising the code below, we can think of it like this: Frame 1 is created with the number 85, which is higher than the target so it terminates. This is repeated until we reach number 2. Frame 1 is created with the number 2, Frame 2 is using the same index as we haven't passed the target, with the value of 4. This is repeated until we reach Frame 32, where we terminate. The rest of the frames terminate because we add to them the next number in the list e.g. Frame 31 is called with 31*2 + 86 > 64 -> Terminates, Frame 30 is called with 30*2 + 86 > 64 -> Terminates, and so on.

def combination_sum(arr, target):
    ans,temp = [],[]
    arr = sorted(list(set(arr)))
    findNumbers(ans, arr, temp, target, 0)
    return ans
 
def findNumbers(ans, arr, temp, target, index):
    if(target == 0):
        ans.append(list(temp))
        return
    for i in range(index, len(arr)):
        if(target - arr[i]) >= 0:
            temp.append(arr[i])
            findNumbers(ans, arr, temp, target-arr[i], i)
            temp.remove(arr[i])

TopSort in DAGs

Theory

In computer science, a topological sort or topological ordering of a directed graph is a linear ordering of its vertices such that for every directed edge uv from vertex u to vertex v, u comes before v in the ordering. For instance, the vertices of the graph may represent tasks to be performed, and the edges may represent constraints that one task must be performed before another; basically, a topological ordering is just a valid sequence for the tasks.

In mathematics, particularly graph theory, and computer science, a directed acyclic graph (DAG) is a directed graph with no directed cycles. That is, it consists of vertices and edges (also called arcs), with each edge directed from one vertex to another, such that following those directions will never form a closed loop. A directed graph is a DAG if and only if it can be topologically ordered, by arranging the vertices as a linear ordering that is consistent with all edge directions. DAGs (and graphs) can be represented in python with dictionaries, where the key represents the node u and the connections can be represented by the key's value as a list.

Problem Statement:
Write the top sort algorithm

What do we know:

  • In topological sorting, we need to put a node before its adjacent nodes.
  • We need to create a stack to store the nodes. Thus, we initialise a visited array of size N to keep the record of visited nodes.
  • We don’t use the the node immediately, but instead, we first recursively call topological sorting for all its adjacent nodes, then push it to a stack.
  • The code below can be be interpreted as followed: We run a loop from 0 till N. If the node is not marked True in the visited array, we call the recursive function for topological sort. In the function, we mark the current node as True in the visited array and run a loop on all the nodes which has a directed edge to the current node. If the node is not marked True in the visited array, we recursively call the topological sort function on the node. Finally, we push the current node in the stack.
